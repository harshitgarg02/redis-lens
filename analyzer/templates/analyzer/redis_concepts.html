What is Redis?

Redis (Remote Dictionary Server) is an open-source, in-memory data structure store used as a database, cache, and message broker.

Key Features:
  • High Performance: In-memory storage for sub-millisecond response times
  • Data Structures: Strings, Lists, Sets, Hashes, Sorted Sets, and more
  • Persistence: Optional disk persistence with RDB and AOF
  • Scalability: Built-in replication and clustering
In-Memory Database

Master-Slave Replication

Master Node

The master node is the primary Redis instance that:

  • Handles all write operations
  • Serves read operations (configurable)
  • Replicates data to slave nodes
  • Maintains the authoritative data set
Slave Nodes (Replicas)

Slave nodes are read-only copies that:

  • Receive data from the master
  • Handle read operations to scale performance
  • Provide data redundancy and backup
  • Can be promoted to master if needed

Redis Sentinel

Redis Sentinel provides high availability for Redis through automatic failover and monitoring.

Monitoring & Detection
  • Continuously monitors master and slave instances
  • Detects when masters are not working properly
  • Sends notifications about Redis instances status
Automatic Failover
  • Promotes a slave to master when master fails
  • Reconfigures other slaves to use the new master
  • Notifies applications about configuration changes
Configuration Provider
  • Acts as authority for service discovery
  • Clients connect to Sentinels to get current master address
  • Provides configuration updates during failovers

Quorum: Multiple sentinels work together to make decisions and avoid false positives.
